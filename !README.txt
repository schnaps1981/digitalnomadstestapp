3. Логика кеширования и загрузки данных из сети
мониторится база данных (кэш) и только она, любые изменения содержимого базы отображаются в recyclerview

произошел запуск приложения:
	если в кеше что-то есть, то это выводится в recyclerview
	если кеш совсем пуст, инициируется загрузка данных с сервера. 
	
приложение работает:

загруженные данные складываются в кэш, монитор базы тут же отображает содержимое на экран.

при скроллинге вниз, в момент приближения к концу кэша, срабатывает триггер, инициирующий подгрузку данных

если номер текущей страницы, которая подгружается с сервера (&page=) больше 5 (по заданию) 
это означает, что данных больше не будет ни из сети ни из кэша, выводится в футере списка сообщение об этом.

если в процессе загрузки данных отваливается сеть, появляется сообщение и кнопка для инициации еще одной попытки загрузки.

4. Особенности
Загруженные из сети данные в кеш складываются только уникальные, уникализация проверяется по всем полям силами движка БД. 
Это нужно для того, чтобы не было дублей данных в кеше


В базе хранится метка времени, когда запись была помещена в таблицу, она сравнивается с текущим временем, и учитывая время жизни кэша, 
он очищается  частично или полностью, время жизни задается в секундах константой DB_CHACHE_LIFETIME_SEC.
